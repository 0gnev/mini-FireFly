# Recommended GitHub Labels

These labels keep issues and pull requests grouped by type, area, and priority.

## Type

- `type: feature`
- `type: bug`
- `type: docs`
- `type: refactor`
- `type: test`

## Area

- `area: core`
- `area: fanout`
- `area: mockprovider`
- `area: observability`
- `area: ci`
- `area: docs`

## Priority

- `priority: high`
- `priority: medium`
- `priority: low`

## Workflow

- `good first issue`
- `portfolio-polish`
- `engineering-task`
